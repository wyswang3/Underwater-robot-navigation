> 文档版本：v1.0

## 🧭 一、概述

Hover H1000 DVL 采用 **PD6 标准协议**（兼容 EPD6 扩展格式）进行通信。  
该协议基于 **ASCII 文本帧**，通过串口或 TCP 网络进行数据传输，可输出设备姿态、速度、距离等关键导航信息。  

本文件面向开发者，说明：
- PD6 协议语句格式与字段定义；
- 上下行通信规则；
- 解析策略与异常处理；
- 在本项目中对应的 Python 模块实现与调用方式。

---

## ⚙️ 二、通信方式

| 通信类型 | 说明 | 默认参数 |
|-----------|------|----------|
| **串口（UART）** | USB 转 RS-232 串口连接 | 波特率 115200，8N1 |
| **以太网（TCP）** | 100M 网口连接 | IP `192.168.2.102` |
| **命令端口** | 下行控制命令 | `10000` |
| **数据端口** | 上行数据输出 | `10001`（或 `10002`） |

### 数据帧格式
- 每条语句以 `:` 开头，以 `\r\n` 结束；
- 字段间用逗号分隔；
- 单位采用 **mm/s、m、°、℃**；
- 状态位采用 `A`（有效）或 `V`（无效）。

---

## 🧩 三、PD6 协议语句定义（Hover H1000）

Hover H1000 默认在 PD6 模式下输出以下 6 类语句。  
若开启水层测速功能，则会增加 `WI/WS/WE/WD` 四类语句。

| 序号 | 语句标识 | 含义 | 坐标系 | 示例 |
|------|------------|--------|----------|--------|
| 1 | `SA` | 系统姿态角 | 设备坐标系 | `:SA, -1.76, 0.07, 322.57` |
| 2 | `TS` | 时间与系统参数 | - | `:TS,20062812352000,35.0,25.13,2.4,1500.0,000` |
| 3 | `BI` | 对底速度（设备体坐标系） | Body | `:BI, 2443, 391, 118, 37,A` |
| 4 | `BS` | 对底速度（船体坐标系） | Ship | `:BS, -1532, 1945, 37,A` |
| 5 | `BE` | 对底速度（地理坐标系） | NED | `:BE, -1532, 1945, -42,A` |
| 6 | `BD` | 对底距离 / 高度 | NED | `:BD, 0.00, 0.00, 0.00, 39.08, 0.00` |

## 📑 四、字段结构与单位说明

### 1. 姿态数据（SA）
```

:SA, ±PP.PP, ±PP.PP, PPP.PP<CR><LF>

```
| 字段 | 含义 | 单位 |
|------|------|------|
| 第1项 | Pitch（俯仰角） | ° |
| 第2项 | Roll（横滚角） | ° |
| 第3项 | Heading（艏向角） | ° |

---

### 2. 时间与系统参数（TS）
```

:TS, YYMMDDHHmmsshh, SS.S, ±TT.T, DDDD.D, CCCC.C, BBB<CR><LF>

```
| 字段 | 含义 | 单位 |
|------|------|------|
| 1 | 时间戳（年月日时分秒×10ms） | 无 |
| 2 | 盐度 | PPT |
| 3 | 水温 | ℃ |
| 4 | 吃水深度 | m |
| 5 | 声速 | m/s |
| 6 | 故障码 + 协议版本 | - |

> 故障码：  
> `0`=正常，`1-4`=波束通道异常。

---

### 3. 对底速度（体坐标系）BI
```

:BI, ±XXXXX, ±YYYYY, ±ZZZZZ, ±EEEEE, S<CR><LF>

```
| 字段 | 含义 | 单位 | 说明 |
|------|------|------|------|
| X | 前向速度 | mm/s | Body X（设备箭头方向） |
| Y | 右向速度 | mm/s | Body Y |
| Z | 向下速度 | mm/s | Body Z |
| E | 速度误差 | mm/s | 系统估计误差 |
| S | 状态标志 | A/V | A=有效，V=无效 |

---

### 4. 对底速度（地理坐标系）BE
```

:BE, ±XXXXX, ±YYYYY, ±ZZZZZ, S<CR><LF>

```
| 字段 | 含义 | 单位 | 坐标轴 |
|------|------|------|------|
| X | 东向速度 | mm/s | East |
| Y | 北向速度 | mm/s | North |
| Z | 向上速度 | mm/s | Up |
| S | 状态标志 | A/V | 有效/无效 |

---

### 5. 对底距离与高度（BD）
```

:BD, +EEEEEEEE.EE, +NNNNNNNN.NN, +UUUUUUUU.UU, DDDD.DD, TTT.TT<CR><LF>

````
| 字段 | 含义 | 单位 | 说明 |
|------|------|------|------|
| E | 东向累计距离 | m | 相对初始点 |
| N | 北向累计距离 | m | 相对初始点 |
| U | 垂向累计距离 | m | 相对初始点 |
| D | 对底高度 | m | 当前距离水底高度 |
| T | 时间间隔 | s | 自上一有效速度帧起的间隔 |

---

## 🔄 五、下行命令协议（控制指令）

所有命令均为 **16 字符 ASCII 字符串**，长度不足补 `0`。

| 命令 | 示例 | 功能 |
|------|--------|------|
| `PR` | `PR 10\r0000000000` | 设置输出频率（1–20 Hz） |
| `BX` | `BX 68.00\r000000` | 设置量程（斜距） |
| `DF` | `DF 0\r00000000000` | 设置输出格式：0=PD6, 1=PLT, 3=EPD6 |
| `BR` | `BR 115200\r000000` | 设置波特率 |
| `IP` | `IP 192.168.2.14\r0000` | 修改 IP 地址 |
| `CS` | `CS \r000000000000` | 启动测量 |
| `CZ` | `CZ \r000000000000` | 停止测量 |

---

## 🧮 六、坐标系定义

| 坐标系 | 说明 | 方向定义 |
|---------|--------|----------|
| **Body (B)** | 设备本体坐标系 | X 向前、Y 向右、Z 向下 |
| **Earth (E/NED)** | 地理坐标系 | X 向东、Y 向北、Z 向上 |
| **安装朝向** | 外壳箭头方向为 X 轴正向 | |
| **角度方向** | 右手定则 | 顺时针为正 |

速度单位为 **mm/s**，在程序中转换为 **m/s**。  
角度以 **度** 表示，内部计算转换为 **弧度**。

## 🧠 七、解析策略（项目实现约定）

**文件位置：** `uwnav/drivers/dvl/hover_h1000/protocol.py`

### 1. 行解析逻辑
```python
if not line.startswith(':'):
    return None
msg_type = line[1:3]
fields = [f.strip() for f in line[3:].split(',')]
````

### 2. 转换规则

| 字段类型     | 处理逻辑                   |
| -------- | ---------------------- |
| 数字（mm/s） | `/ 1000.0` → m/s       |
| 角度（°）    | `np.deg2rad()`         |
| 状态位      | `'A'`→True，`'V'`→False |
| 时间戳      | 由上层统一注入（`time.time()`） |

### 3. 输出数据结构

```python
@dataclass
class DVLData:
    timestamp: float
    vel_body: np.ndarray
    vel_earth: np.ndarray
    height: float
    valid: bool
```

### 4. 容错策略

* 忽略非 `:` 开头行；
* 忽略字段缺失或非法数值；
* 对无效帧 (`S=V`) 不计入融合；
* 打印异常计数 ≤ 5 次后静默。
