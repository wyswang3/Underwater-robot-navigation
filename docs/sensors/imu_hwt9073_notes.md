# IMU（HWT9073-485 / WitMotion）使用注意事项

*文件：`docs/sensors/imu_hwt9073_notes.md`*
*适用：HWT905X / HWT9073 / WT901C485 等 WIT 动态 IMU（RS485 + Modbus）*

---

## 1. 概述

本 IMU 是 9 轴惯性测量单元（加速度 + 角速度 + 地磁 + 姿态解算），通过 **RS485** 以 **Modbus 高速寄存器读取方式** 接入系统。
在本项目中：

* **上位机（OrangePi）使用 230400bps 读取 IMU 数据**
* **使用统一时间基 timebase（MonoNS + EstNS）对齐所有传感器时间**
* **IMU 采用回调 + 线程队列 + Filter 滤波器结构**
* **IMU 是整个导航系统的主时间驱动源（100Hz）**

---

## 2. 安装与硬件注意事项

### 2.1 供电要求

* 输入电压：**5V（USB/TTL）或 12V（485 工业版）**
* 供电必须稳定，严禁使用波动大的电池/开关电源。
* 建议使用 ROV 主电池经 DC-DC 降压稳压后供电。

### 2.2 接线方式

接口类型：**RS485-A / RS485-B**

* A → USB/485 转换器 A
* B → USB/485 转换器 B
* GND → 系统地必须共地（Orange Pi、DVL、Motor Board 均共地）

若 RS485 信号无法识别，请检查：

* 是否 A/B 反接（常见错误）
* 是否未共地（第二常见错误）

### 2.3 安装方向（重要）

IMU 的姿态解算依赖内部算法，对安装角度敏感：

* 本项目使用 **0° 安装角度**（出厂默认）
* 若实际安装方向不同，必须配置：

  * **角度补偿**（可在程序内或设备配置工具中设置 `InstallationAngle`）
* 安装时应尽可能：

  * 固定牢固，无振动
  * 距离强磁源（电机、电调）> 10 cm
  * 避免紧靠高压线缆
  * 避免靠近泵、电磁装置

### 2.4 水下实验注意事项

* IMU 必须置于密封仓内
* 仓体不能产生剧烈磁干扰
* 强电流（推进器）工作时可能会影响磁场 → 推荐 **不在导航融合中使用磁力计**（我们系统已默认关闭）

## 3. 通信与数据读取规范

### 3.1 通信参数

本项目的约定：

* **串口：`/dev/ttyUSB1`**
* **波特率：`230400`**
* **Modbus 地址：`0x50`**
* **数据格式：厂商自定义寄存器映射**

### 3.2 读取寄存器

依赖文件：

* `uwnav/drivers/imu/WitHighModbus/device_model.py`
* `uwnav/drivers/imu/WitHighModbus/i_protocol_resolver.py`

你已经添加了读取四元数 + 线加速度 + 角速度的全部寄存器，解析逻辑已经正确。

---

## 4. 数据预处理与滤波

### 4.1 滤波器结构

实时滤波器 `RealTimeIMUFilter` 包含：

* 1 秒静态校准（漂移估计）
* 加速度/角速度 **6 Hz 低通滤波**
* 航向角（yaw）积分解缠（unwrap）
* 根据时间基 `mono_s` 计算 dt（单调时钟保证连续）

### 4.2 数据格式

每一帧 IMU 数据会被转换为：

```
MonoNS   — 单调时钟（导航系统统一参考时间）
EstNS    — 估计的真实时间（仅用于可读性）
AccX/Y/Z — 加速度（m/s²）
GyrX/Y/Z — 角速度（deg/s）
```

注意：

* 厂商输出单位：**g / deg/s**
* 系统内部统一单位：**m/s² / rad/s**（已自动换算）

## 5. 使用统一时间基（TimeBase）

IMU 是我们系统时间对齐的核心传感器之一。
取样时间调用：

```python
mono_ns, est_ns = stamp("imu0", SensorKind.IMU)
```

用途：

* **MonoNS**：融合算法、ESKF、积分、DVL 对齐
* **EstNS**：写入 CSV，人类可读（类似真实时间）

IMU 回调发生在高频 100Hz，因此使用单调时钟保证无回拨、无跳变、无系统时间漂移影响。

## 6. 程序结构与线程模型

```
IMU 串口线程（底层）
 ↓ 回调 _on_sample()
IMUSample 队列（2000 容量）
 ↓
滤波线程 _loop()
 ↓
写入 raw.csv + filt.csv
```

* 回调线程非常轻量，不做复杂运算
* 所有滤波都在后台线程进行
* 防止高频抖动导致 IO 堵塞
* CSV 写入有定时 flush（0.5 sec）

## 7. 常见问题与排查

### 7.1 偶尔出现 Acc=0 或 Gyr=0

可能原因：

* 串口丢包
* Modbus 帧校验失败
* RS485 电平不足

解决：

* 检查接地
* 降低线缆长度
* 避免与 DVL、泵、电机平行走线

---

### 7.2 姿态角出现跳变

原因：

* 厂商内部算法故障
* 强磁干扰
* 内置低精度地磁影响 yaw

解决：

* 项目中 **不使用 IMU 姿态角**，只使用角速度
* 姿态角由 ESKF（IMU + DVL）重新估算

---

### 7.3 数据时间不同步

如果你的 Log 文件里多个传感器时间不一致：

* 检查是否忘记把传感器改成：
  `mono_ns, est_ns = stamp("imu0", SensorKind.IMU)`
* 检查是否存在 Python 多线程 clock 漂移
* 确认所有程序都使用 `timebase.py`

---

### 7.4 IMU 与 DVL 的时间对不上

解决办法：

* DVL 和 IMU 都使用 **同一个 TimeBase 全局实例**

即：

```python
from uwnav.io.timebase import stamp
```

系统内所有传感器统一时间来源后，不可能再出现 “跨传感器 5–10ms” 的自然漂移。

## 8. 建议的下水实验流程

1. IMU 通电后静置 2–3 秒
2. 观察滤波器是否完成静态校准
3. 轻微摆动设备，检查加速度与角速度是否正常
4. 启动 DVL → IMU → 电机
5. 确保所有传感器记录都已写入统一时间戳
6. 正式下水进行调试

## 9. 文件命名规范

IMU 记录文件示例：

```
imu_raw_20251126_153045.csv
imu_filt_20251126_153045.csv
```

所有文件包含：

* MonoNS、EstNS
* 稳态加速度（校准后）
* γ（yaw）滤波结果

便于与 DVL、深度计等进行融合。

## 10. 结论

通过上述规范，IMU 在系统中扮演：

* **惯性参考主源（100Hz）**
* **导航融合时间基准源**
* **姿态与角速度提供者**
* **ESKF 的高频驱动器**

在 RS485+Modbus 模式、统一时间基、滤波和线程队列机制的保障下，IMU 数据能够稳定、高频、低漂移地投入到后续导航定位中。
