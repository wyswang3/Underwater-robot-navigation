# 香橙派上更新导航项目操作说明（给操作员）

目标：
在香橙派上拉取最新的导航项目 `Underwater-robot-navigation`，切换到分支 `feature/nav-core-online-eskf`，并完成编译。

> 下面所有命令都在香橙派上执行，普通用户账号即可（例如 `orangepi`）。

---

## 1. 准备工作

1. 确认系统有 `git`、`cmake`、`g++` 等基础工具：

   ```bash
   git --version
   cmake --version
   g++ --version
   ```

   若提示 “command not found”，需要先用 `apt` 安装（这个一般由维护人员处理，操作员如果看到报错可以原样反馈给我们）。

2. 约定工作目录：

   建议在家目录下使用统一目录：

   ```bash
   mkdir -p ~/orangepi
   cd ~/orangepi
   ```

---

## 2. 获取 / 更新代码仓库

### 情况 A：第一次在香橙派上拉这个仓库

1. 进入工作目录：

   ```bash
   cd ~/orangepi
   ```

2. 克隆仓库：

   ```bash
   git clone https://github.com/wyswang3/Underwater-robot-navigation.git
   ```

3. 进入仓库目录：

   ```bash
   cd Underwater-robot-navigation
   ```

4. 切换到指定分支：

   ```bash
   git checkout feature/nav-core-online-eskf
   ```

---

### 情况 B：香橙派上已经有旧的 `Underwater-robot-navigation` 目录

1. 进入仓库：

   ```bash
   cd ~/orangepi/Underwater-robot-navigation
   ```

2. 查看当前分支与状态（只是确认，不会改动）：

   ```bash
   git status
   ```

3. 拉取远程最新内容：

   ```bash
   git fetch origin
   ```

4. 切换到新分支并同步远程：

   ```bash
   git checkout feature/nav-core-online-eskf
   git pull origin feature/nav-core-online-eskf
   ```

> 如果 `git checkout` 提示有未提交修改、无法切换，可以先把错误信息截图/复制回来，我们再决定是备份旧目录还是强制丢弃修改。

---

## 3. 编译 nav_core

1. 确保已经在仓库根目录：

   ```bash
   cd ~/orangepi/Underwater-robot-navigation
   ```

2. 进入 `nav_core` 子目录：

   ```bash
   cd nav_core
   ```

3. 创建并进入 `build` 目录：

   ```bash
   mkdir -p build
   cd build
   ```

4. 运行 CMake 生成工程：

   ```bash
   cmake ..
   ```

   * 如果这里报错，请把错误信息完整发回来。
   * 不要自行修改 CMakeLists。

5. 编译：

   ```bash
   make -j4
   ```

   * `-j4` 表示最多用 4 个并行任务，如果板子负载太高，可以改成 `-j2` 或直接 `make`。
   * 编译成功后，`build/bin` 目录下会有两个程序：

     * `uwnav_navd`
     * `uwnav_dvl_selftest`

---

## 4. 运行导航程序（nav_daemon）

1. 回到 `nav_core` 根目录（注意：**不是** build 目录）：

   ```bash
   cd ~/orangepi/Underwater-robot-navigation/nav_core
   ```

2. 启动导航守护进程：

   ```bash
   ./build/bin/uwnav_navd
   ```

3. 程序启动后，如果一切正常，会看到类似输出：

   ```text
   [nav_daemon] starting with config=config/nav_daemon.yaml eskf_config=config/eskf.yaml
   ...
   ```

   * 如果提示找不到 `config/nav_daemon.yaml` 或 `eskf.yaml`，说明当前工作目录不对，确认是在 `nav_core` 目录下启动的。
   * 若有其他错误信息，请原样记录/截图发回来。

---

## 5. 简单检查

更新完成后，操作员可以做两个简单检查：

1. 确认分支确实是我们指定的那个：

   ```bash
   cd ~/orangepi/Underwater-robot-navigation
   git branch
   ```

   输出中应看到类似：

   ```text
     feature/其他分支...
   * feature/nav-core-online-eskf
   ```

   星号标记的就是当前分支。

2. 确认可执行文件存在：

   ```bash
   ls ~/orangepi/Underwater-robot-navigation/nav_core/build/bin
   ```

   里面应至少包含：

   ```text
   uwnav_navd
   uwnav_dvl_selftest
   ```
